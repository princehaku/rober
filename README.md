# rober

`rober` 是一个面向真实生活场景的 **垃圾投递/送达机器人项目**。  
它的目标不是展示复杂技术名词，而是把一件朴素的小事做成稳定体验：

- 用户把垃圾交给机器人；
- 机器人理解任务并安全出发；
- 机器人把垃圾送到指定站点；
- 机器人完成返回或待命，准备下一次服务。

## 我们正在构建的，不只是一个“能跑”的 demo

rober 的长期方向，是成为普通用户也能放心使用的日常机器人能力：

- **更自然的人机协作**：用户用最少步骤完成任务确认，机器人用清晰状态反馈降低不确定感。  
- **更可靠的任务闭环**：从任务发起到送达完成，每个关键阶段都可追溯、可解释、可复盘。  
- **更可持续的工程演进**：功能迭代与验证证据同步沉淀，让系统随着场景增长而持续变强。  
- **更可落地的产品边界**：坚持“送达价值优先”，在安全、成本与体验之间保持可规模化的平衡。

## rober 的高层能力版图

项目围绕一个核心主链路持续打磨：

1. 任务接收与状态编排；
2. 路线执行与阶段切换；
3. 送达确认与异常收敛；
4. 返回/待命与证据留存。

在此基础上，rober 逐步扩展跨楼层与复杂环境下的 assisted delivery 能力，目标是在更多真实场景中保持一致、可验证的服务质量。

## 项目原则

- **价值聚焦**：核心价值是“把垃圾送到该去的地方”，不是炫技式功能堆叠。  
- **证据优先**：所有完成度判断以可复盘证据为准，避免口头“已完成”。  
- **边界清晰**：阶段性能力与长期愿景分层表达，不把规划当成已交付结果。  
- **持续迭代**：通过 sprint 留档、验证和复盘，把每一轮改进沉淀为下一轮起点。

## 愿景

我们希望 rober 最终成为一个“对普通人友好、对工程团队可维护、对真实环境可演进”的机器人产品基座。  
今天的每一步都在为这个目标积累：让机器人从“技术可行”，走向“用户可用、可依赖、可持续交付”。
